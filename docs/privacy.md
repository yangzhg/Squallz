# Squallz 隐私说明

发布前需要随安装包和官网再次复核。

## 核心承诺

- 无广告、无捆绑、无遥测。
- 不上传压缩包内容、文件名、路径、密码、恢复数据或操作历史。
- 崩溃报告只会在后续实现 opt-in 流程后由用户主动开启；默认关闭。当前首发开发线没有启用自动崩溃上报。

## 本地保存的数据

- 设置：主题、语言、UI 模式、强调色、安全阈值、性能选项等。
- 历史记录：最近打开或最近操作的本地路径，仅用于应用内体验。
- 密码：只有用户显式选择记住当前压缩包密码时，才写入系统密码库。当前后端覆盖 macOS Keychain、
  Linux Secret Service（通过 `secret-tool`）和 Windows Credential Manager；每个平台都必须用对应
  smoke 在真实目标系统验收后才能作为正式发布证据。
  Squallz 不把明文密码写入设置、localStorage、日志或任务队列。
- 诊断报告：本地诊断输出不得包含明文密码或压缩包内容。

## 网络访问

当前产品功能不需要把用户文件发送到网络服务。未来如果加入自动更新、可选崩溃报告或帮助站链接，
必须在发布说明和本页中说明触发条件、发送内容和关闭方式。

## 外部工具

某些格式可能需要本机安装的外部工具，例如 7zz/7z、wimlib-imagex 或 par2。Squallz 只通过本机进程调用
这些工具处理用户指定的本地文件；是否随包分发这些工具取决于许可证和发布检查。

## 用户控制

- 可以在设置页关闭历史相关体验或清理本地记录。
- 可以在密码弹窗或密码本入口忘记某个压缩包的已保存密码。
- 可以用 CLI 的 `--json` 输出做自动化，同时避免解析面向人的文本。
